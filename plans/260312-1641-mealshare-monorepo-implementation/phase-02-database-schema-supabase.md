# Phase 02: Database Schema & Supabase

## Context Links
- Parent: [plan.md](./plan.md)
- Research: [Stack Research](./research/stack-research-turborepo-next-nestjs-supabase-integration.md), [Realtime Research](./research/researcher-02-realtime-chat-financial-voting-deployment.md)
- Dependencies: Phase 01 (monorepo must exist)

## Overview
Define complete Prisma schema, run migrations against Supabase PostgreSQL, enable Realtime on `group_messages`, configure RLS policies, and set up Supabase Storage for image uploads.

## Key Insights
- Prisma connects directly to Supabase PostgreSQL connection string
- NestJS uses service role key (bypasses RLS) for admin operations
- RLS policies enforce chat access and order visibility at database level
- Supabase Realtime needs explicit table enable in dashboard/SQL
- Unique constraint on `(vote_id, user_id)` prevents duplicate votes
- One confirmed menu per `(group_id, week_start_date)` via partial unique index

## Requirements

### Functional
- All 11 tables created with proper relationships and indexes
- RLS policies for `group_messages` (active members only)
- RLS policies for `daily_orders` (own records only)
- Supabase Realtime enabled on `group_messages`
- Supabase Storage bucket for menu item images

### Non-functional
- Migration reproducible via `prisma migrate`
- UUID primary keys (Supabase standard)
- Timestamps with timezone
- Soft-delete pattern for messages

## Architecture

### Entity Relationship Summary
```
users ──< group_members >── groups
users ──< daily_orders
users ──< vote_responses >── vote_options >── votes
groups ──< weekly_menus ──< weekly_menu_items >── menu_items
groups ──< group_messages
groups ──< votes
daily_orders >── weekly_menu_items
```

## Implementation Steps

1. **Initialize Prisma in NestJS app**
   ```bash
   cd apps/api
   pnpm prisma init
   ```
   Update `DATABASE_URL` in `.env` to Supabase connection string

2. **Define Prisma schema** (`apps/api/prisma/schema.prisma`)
   ```prisma
   generator client {
     provider = "prisma-client-js"
   }
   datasource db {
     provider = "postgresql"
     url      = env("DATABASE_URL")
   }

   enum GlobalRole {
     admin
     user
   }
   enum GroupMemberRole {
     leader
     participant
   }
   enum GroupMemberStatus {
     active
     removed
   }
   enum WeeklyMenuStatus {
     draft
     confirmed
   }
   enum VoteStatus {
     open
     closed
     results
   }
   enum MessageType {
     user
     system
   }

   model User {
     id           String   @id @default(uuid()) @db.Uuid
     email        String   @unique
     name         String
     passwordHash String?  @map("password_hash")
     role         GlobalRole @default(user)
     avatarUrl    String?  @map("avatar_url")
     fcmToken     String?  @map("fcm_token")
     googleId     String?  @unique @map("google_id")
     createdAt    DateTime @default(now()) @map("created_at")
     // relations
     ledGroups      Group[]         @relation("GroupLeader")
     memberships    GroupMember[]
     createdMenus   MenuItem[]
     createdWeeklyMenus WeeklyMenu[]
     createdVotes   Vote[]
     voteResponses  VoteResponse[]
     dailyOrders    DailyOrder[]
     sentMessages   GroupMessage[]
     notifications  Notification[]
     @@map("users")
   }

   model Group {
     id          String   @id @default(uuid()) @db.Uuid
     name        String
     description String?
     leaderId    String   @map("leader_id") @db.Uuid
     inviteCode  String   @unique @map("invite_code")
     createdAt   DateTime @default(now()) @map("created_at")
     leader      User     @relation("GroupLeader", fields: [leaderId], references: [id])
     members     GroupMember[]
     weeklyMenus WeeklyMenu[]
     votes       Vote[]
     dailyOrders DailyOrder[]
     messages    GroupMessage[]
     @@map("groups")
   }

   model GroupMember {
     id        String            @id @default(uuid()) @db.Uuid
     groupId   String            @map("group_id") @db.Uuid
     userId    String            @map("user_id") @db.Uuid
     role      GroupMemberRole   @default(participant)
     status    GroupMemberStatus @default(active)
     joinedAt  DateTime          @default(now()) @map("joined_at")
     removedAt DateTime?         @map("removed_at")
     group     Group  @relation(fields: [groupId], references: [id])
     user      User   @relation(fields: [userId], references: [id])
     @@unique([groupId, userId])
     @@map("group_members")
   }

   model MenuItem {
     id          String   @id @default(uuid()) @db.Uuid
     name        String
     description String?
     imageUrl    String?  @map("image_url")
     category    String?
     createdBy   String   @map("created_by") @db.Uuid
     createdAt   DateTime @default(now()) @map("created_at")
     updatedAt   DateTime @updatedAt @map("updated_at")
     creator     User     @relation(fields: [createdBy], references: [id])
     weeklyMenuItems WeeklyMenuItem[]
     voteOptions     VoteOption[]
     @@map("menu_items")
   }

   model WeeklyMenu {
     id            String          @id @default(uuid()) @db.Uuid
     groupId       String          @map("group_id") @db.Uuid
     weekStartDate DateTime        @map("week_start_date") @db.Date
     status        WeeklyMenuStatus @default(draft)
     createdBy     String          @map("created_by") @db.Uuid
     createdAt     DateTime        @default(now()) @map("created_at")
     group         Group    @relation(fields: [groupId], references: [id])
     creator       User     @relation(fields: [createdBy], references: [id])
     items         WeeklyMenuItem[]
     @@unique([groupId, weekStartDate, status])
     @@map("weekly_menus")
   }

   model WeeklyMenuItem {
     id           String     @id @default(uuid()) @db.Uuid
     weeklyMenuId String     @map("weekly_menu_id") @db.Uuid
     menuItemId   String     @map("menu_item_id") @db.Uuid
     price        Decimal    @db.Decimal(10,2)
     isAvailable  Boolean    @default(true) @map("is_available")
     weeklyMenu   WeeklyMenu @relation(fields: [weeklyMenuId], references: [id])
     menuItem     MenuItem   @relation(fields: [menuItemId], references: [id])
     dailyOrders  DailyOrder[]
     @@map("weekly_menu_items")
   }

   model Vote {
     id            String     @id @default(uuid()) @db.Uuid
     groupId       String     @map("group_id") @db.Uuid
     title         String
     weekStartDate DateTime   @map("week_start_date") @db.Date
     createdBy     String     @map("created_by") @db.Uuid
     endsAt        DateTime   @map("ends_at")
     status        VoteStatus @default(open)
     createdAt     DateTime   @default(now()) @map("created_at")
     group         Group      @relation(fields: [groupId], references: [id])
     creator       User       @relation(fields: [createdBy], references: [id])
     options       VoteOption[]
     responses     VoteResponse[]
     @@map("votes")
   }

   model VoteOption {
     id         String  @id @default(uuid()) @db.Uuid
     voteId     String  @map("vote_id") @db.Uuid
     menuItemId String  @map("menu_item_id") @db.Uuid
     vote       Vote    @relation(fields: [voteId], references: [id])
     menuItem   MenuItem @relation(fields: [menuItemId], references: [id])
     responses  VoteResponse[]
     @@map("vote_options")
   }

   model VoteResponse {
     id           String     @id @default(uuid()) @db.Uuid
     voteId       String     @map("vote_id") @db.Uuid
     userId       String     @map("user_id") @db.Uuid
     voteOptionId String     @map("vote_option_id") @db.Uuid
     createdAt    DateTime   @default(now()) @map("created_at")
     vote         Vote       @relation(fields: [voteId], references: [id])
     user         User       @relation(fields: [userId], references: [id])
     voteOption   VoteOption @relation(fields: [voteOptionId], references: [id])
     @@unique([voteId, userId])
     @@map("vote_responses")
   }

   model DailyOrder {
     id              String       @id @default(uuid()) @db.Uuid
     userId          String       @map("user_id") @db.Uuid
     groupId         String       @map("group_id") @db.Uuid
     date            DateTime     @db.Date
     weeklyMenuItemId String      @map("weekly_menu_item_id") @db.Uuid
     quantity        Int          @default(1)
     totalPrice      Decimal      @map("total_price") @db.Decimal(10,2)
     createdAt       DateTime     @default(now()) @map("created_at")
     user            User         @relation(fields: [userId], references: [id])
     group           Group        @relation(fields: [groupId], references: [id])
     weeklyMenuItem  WeeklyMenuItem @relation(fields: [weeklyMenuItemId], references: [id])
     @@unique([userId, groupId, date, weeklyMenuItemId])
     @@map("daily_orders")
   }

   model GroupMessage {
     id        String      @id @default(uuid()) @db.Uuid
     groupId   String      @map("group_id") @db.Uuid
     senderId  String?     @map("sender_id") @db.Uuid
     content   String
     type      MessageType @default(user)
     createdAt DateTime    @default(now()) @map("created_at")
     editedAt  DateTime?   @map("edited_at")
     deletedAt DateTime?   @map("deleted_at")
     group     Group  @relation(fields: [groupId], references: [id])
     sender    User?  @relation(fields: [senderId], references: [id])
     @@map("group_messages")
   }

   model Notification {
     id        String   @id @default(uuid()) @db.Uuid
     userId    String   @map("user_id") @db.Uuid
     title     String
     body      String
     type      String
     data      Json?
     readAt    DateTime? @map("read_at")
     createdAt DateTime  @default(now()) @map("created_at")
     user      User @relation(fields: [userId], references: [id])
     @@map("notifications")
   }
   ```

3. **Run initial migration**
   ```bash
   cd apps/api
   pnpm prisma migrate dev --name init
   ```

4. **Create PrismaService** (`apps/api/src/prisma/prisma.service.ts`)
   ```typescript
   @Injectable()
   export class PrismaService extends PrismaClient implements OnModuleInit {
     async onModuleInit() { await this.$connect(); }
   }
   ```
   Create `PrismaModule` as global module.

5. **Create seed script** (`apps/api/prisma/seed.ts`)
   - Insert admin user (email: admin@mealshare.app, hashed password)
   - Insert sample menu items (5-10 dishes with categories)
   - Add to `package.json`: `"prisma": { "seed": "ts-node prisma/seed.ts" }`

6. **Enable Supabase Realtime** (`supabase/enable-realtime.sql`)
   ```sql
   ALTER PUBLICATION supabase_realtime ADD TABLE group_messages;
   ```

7. **Create RLS policies** (`supabase/rls-policies.sql`)
   ```sql
   -- Enable RLS
   ALTER TABLE group_messages ENABLE ROW LEVEL SECURITY;
   ALTER TABLE daily_orders ENABLE ROW LEVEL SECURITY;

   -- Chat: active members can SELECT
   CREATE POLICY "active_members_read_messages" ON group_messages FOR SELECT USING (
     EXISTS (
       SELECT 1 FROM group_members
       WHERE group_id = group_messages.group_id
         AND user_id = auth.uid() AND status = 'active'
     )
   );

   -- Orders: users see own records
   CREATE POLICY "users_read_own_orders" ON daily_orders FOR SELECT USING (
     user_id = auth.uid()
   );
   ```
   Note: NestJS uses service role key (bypasses RLS) for all write operations.

8. **Set up Supabase Storage**
   - Create bucket `menu-images` in Supabase dashboard
   - Set public access policy for reading images
   - Upload via NestJS using `@supabase/supabase-js` with service role key

9. **Add Supabase client to NestJS**
   ```bash
   cd apps/api && pnpm add @supabase/supabase-js
   ```
   Create `SupabaseService` wrapping `createClient` with service role key.

10. **Export types to packages/types**
    - Generate Prisma types, re-export relevant interfaces in `packages/types/src/index.ts`
    - Export enums: `GlobalRole`, `GroupMemberRole`, `WeeklyMenuStatus`, `VoteStatus`, `MessageType`

## Risk Assessment
- **Supabase connection pooling**: Use `?pgbouncer=true&connection_limit=1` for serverless; direct connection for NestJS
- **RLS + service role**: Ensure NestJS always uses service role key; frontend uses anon key
- **Migration conflicts**: Keep single migration source of truth in Prisma

## Security Considerations
- Service role key only in NestJS environment (never exposed to frontend)
- RLS policies as defense-in-depth (NestJS guards are primary access control)
- Password hashes stored using bcrypt (Phase 03)
- UUID primary keys prevent enumeration attacks
